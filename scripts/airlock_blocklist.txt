# airlock_blocklist.txt
# ---------------------------------------------------------------------------
# Default strategy-term blocklist for the FUSION-lane airlock check
# (`scripts/airlock_check.py`).
#
# Purpose:
#   The FUSION lane keeps ALL strategy content in the `.fusion.json` companion
#   and NONE of it in the `.txt` sketch. The airlock check enforces this by
#   scanning the .txt for substrings that look like strategy. Anything matched
#   here in the .txt (outside of comment/source lines) causes a REJECT.
#
# Format:
#   - One token per line
#   - Lines starting with `#` are comments
#   - Blank lines ignored
#   - Tokens are matched CASE-INSENSITIVELY, as whole-word boundaries on the
#     normalized .txt content (see scripts/airlock_check.py for matching rules)
#
# Maintenance:
#   - This file is the DEFAULT blocklist. Authors may pass `--blocklist <path>`
#     to airlock_check.py for problem-specific lists.
#   - Additionally, every NAMED TECHNIQUE / BRIDGE LEMMA term used inside the
#     companion `.fusion.json` is treated as an *implicit* blocklist entry
#     for that specific submission. Authors do NOT need to update this file
#     when they pick a new named technique — the airlock derives the dynamic
#     blocklist from the companion automatically.
#
# Reference:
#   docs/fusion_axis_companion_spec.md
#   docs/infrastructure_changes_may30/I8_fusion_lane.md
# ---------------------------------------------------------------------------
